## 1. 概要・ビジョン

企業名で検索すると、決算書・有価証券報告書・財務諸表を**自動で読みに行き**、財務3表を可視化し、
主要指標を算出し、そこに**AIによる「決算書の読み方の解説」と「バランスシートの健全性分析」**を重ねて返す
オンラインプラットフォーム。「開示資料は公開されているのに、素人には読めない」というギャップを、
**自動取得 × 可視化 × AIの平易な解説**で埋める。

一言で言えば「**決算書を、誰でも読めるようにするAIアナリスト**」。PDF.ai（開示×AI）の姉妹プロダクト。

## 2. 対象ユーザーと提供価値

| ユーザー | 悩み | 本サービスの価値 |
|---|---|---|
| 個人投資家・株主（主） | 有報・決算短信を開いても、どこを見ればいいか分からない／同業比較が手間 | 検索一発で3表可視化＋指標＋「この会社の財務はこう読む」というAI所見 |
| 就活・転職者（副） | 志望企業の実態（安定性・成長性）を知りたい | 平易な言葉での財務の健康診断・成長トレンド |
| 会計/経営の学習者（副） | 決算書の読み方を実例で学びたい | 実際の企業データに紐づく「読み方解説」＝生きた教材 |

**投資助言はしない**（§8参照）。あくまで「公開情報の整理・可視化・教育的解説」に徹する。

## 3. 主要機能

### MVP（最小の一気通貫）
1. **企業検索**: 企業名・証券コードで検索 → EDINET から対象企業・最新の有報/四半期報告を特定。
2. **開示資料の自動取得**: EDINET API v2 で XBRL（財務データ）を取得・パース。
3. **財務3表の可視化**: 貸借対照表（BS）・損益計算書（PL）・キャッシュフロー計算書（CF）を整形表示＋主要項目のグラフ。
4. **主要指標の自動算出**: 自己資本比率・流動比率・当座比率・ROE・ROA・売上高/営業利益率・営業CFなど。各指標に「良否の目安」を併記。
5. **決算書の読み方の解説（AI）**: その企業の数字に即して、「BS/PL/CF の各表はどこを見るか」を Gemini が平易に解説。
6. **BS健全性の分析（AI）**: 自己資本の厚み・負債構成・流動性・運転資本などから「財務の健康診断」所見を生成（断定でなく所見＋根拠）。

### 客観指標・AI分析（本プロダクトの核 / 2026-07-06 Keita 追加方針）
「助言はしない」を守りつつ、**客観的に算出できる指標は示す**。AIは推奨でなく「分析・整理」を担う。
7. **バリュエーション（適正化）**: PER・PBR・EV/EBITDA・配当利回りを算出し、**過去レンジ・同業平均との相対で「割安/妥当/割高」を客観表示**（推奨ではなく事実としてのレンジ位置）。
8. **理論株価（参考値）**: 複数モデル（DCF・配当割引DDM・PER/PBRベースのマルチプル）で**前提を明示した参考レンジ**を機械的に算出し、現在株価と並置。「あくまでモデル前提に基づく参考値・投資判断は自己責任」と明記（§8）。
9. **健全性スコア**: 収益性・安全性・成長性・キャッシュ創出の各観点をスコア化し、総合の「財務の健康診断」を客観提示（Simply Wall St の発想）。
10. **AIの視点による所見（グラウンディング必須）**: 「**ここが気になります**」「**ここはこうなっています**」という観点で、AIが分析・整理する。**最重要要件＝各所見を財務諸表の具体箇所に紐づける**（例:「有利子負債の増加は BS・流動負債の短期借入金に表れています」「利益率低下は PL・売上原価率の上昇として表れています」）。所見には必ず「どの表・どの科目・何期比」の出典タグを付す。
11. **競合他社比較**: 同業他社と並べ、指標・成長性・財務構成の**特徴の違いを整理**（「他社より自己資本が厚い／在庫回転が遅い」等）＋その差が財務諸表のどこに表れるかを言及。

> **設計原則（グラウンディング）**: すべての数値はコードで確定し原典（有報の該当箇所）にリンク。AIの所見は必ず確定済み数値・具体科目に紐づけ、数字の生成はさせない（幻覚防止＋説明可能性）。

### 拡張（フェーズ2以降）
- **経年トレンド**: 5〜10期の推移グラフ（売上/利益/自己資本比率など）。
- **同業比較**: 同業種・同規模企業との指標比較（EDINET 業種コード活用）。
- **セグメント/注記の要約**: 有報のテキスト（事業等のリスク・MD&A 相当）を AI 要約。
- **ウォッチリスト/アラート**: 新規開示の通知。
- **米国企業対応**: SEC EDGAR（XBRL companyfacts）連携。
- **PDF アップロード解析**: EDINET 未収録資料は PDF.ai の技術で読み取り（PDF.ai とのシナジー）。

## 4. データ源・技術構成

- **一次データ**: 金融庁 **EDINET API v2**（2026年6月仕様更新・無料APIキー）。書類一覧API＋書類取得API（XBRL/PDF）。閲覧期間の制約に注意し、取得済みデータは自前でキャッシュ/保存。
- **XBRL パース**: XBRL/インラインXBRL から勘定科目をタクソノミに沿って抽出 → 正規化した財務データモデルへ。
- **AI 解説**: Gemini（既存 .mc.env の GEMINI_API_KEY）。数値を渡し、読み方解説・BS所見・要約を生成。数値の算出はコード側で確定し、AIは「解釈・言語化」に限定（幻覚防止）。
- **スタック**: PDF.ai/Apollo を流用（backend Node22+Express+TS / frontend React18+Vite+Tailwind）。公開配信・レート制限・アクセス解析・多言語基盤は PDF.ai の既存部品を再利用。
- **ホスティング**: 独自ドメイン（要取得）＋ cloudflared トンネル、公開ゲート（PDF.ai の publicSite 許可リスト方式を踏襲）。

## 6. 市場・競合と差別化

既存（日本）: バフェット・コード（10期可視化・一部有料）、IR BANK（詳細な財務履歴・広告/有料）、株探（速報・広告/有料）、マネックス銘柄スカウター（証券口座向け・高品質）。海外: Simply Wall St（企業ごとに平易な自動分析＋ビジュアル＝最も近い発想）、stockanalysis.com。

**差別化**:
- **AIネイティブな「読み方解説」**: 単なる数値表示でなく、その企業の数字に即して「どう読むか」を毎回言語化。既存は数値可視化止まりが多い。
- **教育×分析の両取り**: 投資家の深掘りと、初学者の学習を同一画面で。
- **日本語ファースト＋無料公開**: 銘柄スカウターは証券口座必須、他は有料の壁。まず無料で入口を広げる（PDF.ai と同じ戦略）。
- **PDF.ai とのシナジー**: 開示PDFの読み取り資産を持つ。

## 8. 法務・コンプライアンス（重要）

- **EDINET データ利用**: 公共データ利用規約（PDL1.0）に準拠し**商用の二次利用可**。ただし**出典記載が必須**（「出典: 金融庁 EDINET」等）。閲覧期間・提供条件は API 仕様書に従う。
- **金融商品取引法（投資助言業の回避）**: 特定個人向けの投資助言（この銘柄を買え/売れ）は**投資助言・代理業の登録**が必要になる。本サービスは**公開情報の整理・可視化・一般的な教育解説に徹し、個別の売買推奨はしない**。AI出力も「所見・一般論」に留め、投資判断は利用者責任である旨を明示。
- **理論株価・バリュエーションの扱い**: 理論株価や割安/割高は「**モデルと前提を開示した客観的な計算結果**」として提示し、**特定個人への売買推奨・目標株価の助言としては提示しない**。前提（割引率・成長率等）を必ず併記し、複数モデルのレンジで示す（単一の断定値にしない）。「参考値であり投資判断は自己責任」を明示。これで「客観指標の表示」と「投資助言業の回避」を両立する（2026-07-06 Keita 意図＝助言ではなく客観表示）。
- **免責・正確性**: データはベストエフォート、算出やAI解説に誤りがあり得る旨を明記。原典（有報）へのリンクを必ず併設し、利用者が一次情報を確認できるようにする。
- **プライバシー/広告**: PDF.ai で整備済みのプライバシーポリシー・Cookie同意（EEA向けCMP）・第一者解析を流用。

## 8.5 UI／デザイン方針（Keita こだわり・海外参考 / 2026-07-06）

UI は最重要のこだわりポイント。海外の金融データサービスの「情報量が多くても直感的」なデザインを参考にする。

- **参考サイト**: Simply Wall S（健全性の“スノーフレーク”＋平易な自動narrative）／stockanalysis.com（軽快で見やすい指標表・チャート）／Koyfin・Bloomberg・Morningstar（プロ級のデータ密度）／Finchat/Fiscal.ai（AI×財務）。
- **画面構成（企業分析ページ）**: ①ヘッダ＝企業検索＋企業名/業種/現在株価。②**バリュエーション帯**＝理論株価レンジ vs 現在値のゲージ（割安〜割高）。③**健全性スコア**＝総合＋収益性/安全性/成長性/CF のサブスコア（レーダー or スノーフレーク）。④**財務3表ビジュアル**＝BS構成バー・PL推移・CFウォーターフォール。⑤**AIの所見カード**＝「気になる点／現状」を色分けカードで、各カードに“BS・流動負債”等の**出典タグ**と原典リンク。⑥**競合比較**＝同業テーブル＋差分ハイライト。
- **トーン**: クリーン・データビジュアル主体・余白広め・数値は等幅で読みやすく。ダーク/ライト両対応。モバイルでも崩れないレスポンシブ（PDF.ai で整備済みの基盤を流用）。
- **プロセス**: 実装前に**静的モックアップ（HTML）を作って Keita レビュー**→合意した見た目で実装（UI 手戻り防止）。まず企業分析メイン画面のモックを提示する。

## 10. リスクと対策

| リスク | 対策 |
|---|---|
| XBRLタクソノミの企業差・年度差でパース不整合 | 主要科目のマッピング表を持ち、未対応は「取得不可」と明示（黙って誤値を出さない） |
| AIの数値幻覚 | 数値算出はコードで確定、AIは解釈のみ。出力を数値と突き合わせ検証 |
| 投資助言業への抵触 | §8の線引き（推奨しない・教育/情報提供に徹する）＋免責の徹底 |
| EDINET 利用条件・出典記載漏れ | 出典表記を全ページに固定、規約を定期確認 |
| 既存競合の厚み | AI解説×教育×無料×日本語で入口を差別化、PDF.aiユーザー導線 |
