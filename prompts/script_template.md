# 脚本テンプレート

LLM はこのテンプレに従って Markdown を生成する。
人間が `data/scripts/` で目視チェックし、OK なら `data/approved/` に同名ファイルを作る。

## 構成 (合計 15 秒)

| セクション | 秒数 | 役割 |
|---|---|---|
| HOOK | 1.5 | 視聴を止める一言 (数字・反直感) |
| 問題提起 | 3.0 | 視聴者が共感する場面描写 |
| 研究紹介 | 4.5 | 出典・方法・サンプル数 |
| 結論 | 3.0 | 意外な発見の核心 |
| 示唆 | 3.0 | 「明日からどうする」 |

## 出力フォーマット (Markdown)

```markdown
---
paper_id: <openalex_id 等>
title: <論文タイトル>
authors: <著者>
year: <年>
source: <ジャーナル / NBER WP 番号>
url: <doi or url>
duration_seconds: 15
---

# フック
<10〜20 字>

# キャプション (投稿文・style_guide に従う)
...

# シーン

## Scene 1 (0.0-1.5s) — HOOK
- VISUAL: <ミーアキャット博士の演技 + 背景>
- VOICE_JA: "<セリフ>"
- ON_SCREEN_TEXT: "<字幕>"

## Scene 2 (1.5-4.5s) — 問題提起
...

## Scene 3 (4.5-9.0s) — 研究紹介
...

## Scene 4 (9.0-12.0s) — 結論
...

## Scene 5 (12.0-15.0s) — 示唆
...

# 出典の正確な引用
<著者, 年, タイトル, 媒体, DOI>

# Fact-check メモ
- <論文中の数値の引用箇所>
- <断定を避けるべき箇所>
```

## 守るべきルール

- 数字はアブストに書かれた値だけを使う。推定で水増ししない
- 「〜という研究もあります」ではなく「〇〇大学の研究で〜」と具体に
- 因果と相関の取り違えに注意 (観察研究なら「関連が示された」)
- ミーアキャット博士の口調は character_meerkat_phd.md に従う
